You are an expert backend engineer. Build a modular Python backend system for a semiconductor product alternative search engine.
 
The system will ingest locally stored HTML product pages, extract specifications, store them in Oracle database, generate embeddings using OpenAI, and find alternative semiconductor products using a hybrid search architecture.
 
Important Constraints
 
1. No external scraping is allowed.
2. HTML files are already stored locally.
3. A CSV file contains product name, category, and HTML file path.
4. Oracle Database must be used.
5. OpenAI API will be used only for embeddings and reasoning.
6. The system must support multiple semiconductor categories (microcontroller, sensor, power IC, etc.).
7. The system must use a configuration-driven approach for category-specific attributes.
 
---
 
System Architecture
 
Data Ingestion Pipeline
 
CSV → HTML Loader → HTML Parser → Spec Normalizer → Oracle DB → Embedding Generator
 
Search Pipeline
 
User Query → Structured Filter → Vector Similarity → Result Ranking
 
---

---
 
Embedding Generation
 
Use OpenAI embeddings API.
 
Example feature text:
 
"ARM Cortex-M3 microcontroller with 64KB flash, 20KB RAM, 48 pins, SPI I2C UART interfaces, operating voltage 2.0 to 3.6 volts."
 
Store the embedding vector in Oracle.
 
---
 
Hybrid Search Algorithm
 
When user searches for alternatives:
 
Step 1
Find base product in database.
 
Step 2
Apply structured filtering using SQL.
 
Example rules:
 
same category
flash >= base flash
ram >= base ram
pins >= base pins
same architecture (when applicable)
 
Step 3
Perform vector similarity search using embedding column.
 
Step 4
Return top N most similar products.
 
---
 
FastAPI Endpoints

Reads CSV, parses HTML, stores normalized specs in Oracle.
 
/generate-embeddings
Creates embeddings for stored products using OpenAI.
 
/find-alternatives
Input: product name
Output: ranked list of alternative semiconductor products.
